◆ evictionCheckInterval

std::chrono::seconds Atlas::FileManager::Options::evictionCheckInterval { ATLAS_DEFAULT_JANITOR_CHECK_INTERVAL }

How often should the FileJanitor should check for inactive files in seconds. This is set to ATLAS_DEFAULT_JANITOR_CHECK_INTERVAL by default. Please refer to the note below for an important piece of advice regarding what to set this value to.

Note
As a piece of advice, this should be a multiple of fileTTL to ensure that the file is not staying loaded for longer than fileTTL. For example, if fileTTL is 10 seconds, then evictionCheckInterval should be set to 1, 2, 5 or 10 seconds. Once the file is inactive for 10 seconds, it will be unloaded right after the file's lifespan is over, because evictionCheckInterval is 10 seconds.

◆ maxFileCount

size_t Atlas::FileManager::Options::maxFileCount { (size_t)(-1)}

The maximum number of files that can be loaded at once. This is set to -1 by default, which indicates there is no limit. If you want to set a limit, set this value to a positive number.

Note
If this limit is reached, the janitor will unload the least recently used file. This should be set to -1 for most cases, unless there is a specific reason to limit the number of files. This number should be set to a positive number if you want to limit the number of files.

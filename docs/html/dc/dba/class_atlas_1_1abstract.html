Detailed Description

template<typename ConsoleSink>
class Atlas::abstract< ConsoleSink >

This is NOT what will print to the terminal, rather this is an interface where the logger will pass logging messages to. Use this for in-game consoles or something like that.

A class that acts as, well, a gate! It is essentially as a piece of machinery on an assembly line. Please do note, that this is a template class and is not meant to be used directly. Please see GameObjectRegistry for more information and an example.

A drawable is anything that can be drawn to the screen. However, this does not include shaders as that is an entirely seperate system.

Same thing as a gate, but for game objects.

Represents an object that can be rendered in the screen. This is an abstract class that contains all the common properties of a game object.

You may implement this yourself to create your own game objects and drawing code, or you can inherit from Atlas::GameObject to use the default implementation. However, if you inherit from this class, be sure you have a basic understanding of Raylib's and OpenGL's rendering system. If not, see https://learnopengl.com/ and https://www.raylib.com/ for a good place to start. In addition, it is not advised for you to inherit this yourself because it will require you to implement rendering code. However, it is recommended if your game requires a rendering system that is not provided by default.

The IGameObject class is abstract and cannot be instantiated directly. Instead, you should inherit from this class and implement the pure virtual functions. As of version 0.0.3, the pure virtual functions are: render(), texture(), update(), prePhysicsUpdate(), postUpdate(), destroy(), and cleanup().

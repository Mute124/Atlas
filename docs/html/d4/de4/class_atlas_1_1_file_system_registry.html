Detailed Description

The file system registry.

The file system registry is a relatively complex class because it:

  • Keeps track of files.
  • Handles loading of files
  • Handles unloading of files
  • Handles registration of files
  • And just about everything else that a registry does

<par> As an example of how to add a new file extension, see the File Loading document.

◆ registerFile()

void Atlas::FileSystemRegistry::registerFile ( std::string const &  path)

Registers a file into the file registry.

This function creates a new FileMeta object from the provided path, generates a unique index for the file, and adds it to the registry's lookup table. If the file's extension is in the load on register extensions set, it will be loaded immediately. This function is mostly used internally within the FileSystemRegistry::init function during the file registration process.

Parameters
pathThe path to the file that should be registered. This is the exact path to the file, not the file name.

Tells the registry which extensions should be loaded on register. If it exists in the set, it will be loaded.

This is set PRIOR to the discovery of files because this defines what extensions should be immediately loaded on registration. ◆ mLookupTable

std::unordered_map<std::string, uint16_t> Atlas::FileSystemRegistry::mLookupTable
private

An unordered map that maps a file path to a file index. This essentially serves as a reverse lookup table that maps a key to an index.

The key can be either the file path, the file name, the file extension, or really anything that can be used to identify a file. However, Atlas only uses file path, the file name, and the file extension to identify a file. This is the first step in the process of looking up a file. The uint16_t index corresponds to the index in the mFileMetaRegistry unordered map.

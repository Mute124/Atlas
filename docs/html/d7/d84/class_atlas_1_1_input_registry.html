Detailed Description

The input registry is used to register InputActions (InputAction) and their callbacks by using a Laissez-faire system.

Note
Remember that this is a singleton, so you may use this statically.
Bug:
When using functions that require rendering, the engine will crash because the input registry runs on a different thread. This can be fixed through a sync system.
Since
v0.0.6

void Atlas::InputRegistry::unregisterAction ( InputAction *  action)

Unregisters ALL callbacks for a given InputAction.

This is useful if a keybind is no longer needed and you want to remove all callbacks in that category from the input system. For example, this is useful when the player is joining a game. See InputAction for more information on what an InputAction is.

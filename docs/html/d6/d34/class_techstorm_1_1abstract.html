Detailed Description

template<typename K, typename D, typename T>
class Techstorm::abstract< K, D, T >

This is an abstract interface singleton that all projects must implement and set the ProjectReference singleton to their project. If this boilerplate code is not done, Techstorm will not know what to run. In terms of functionality, this interface just holds the functions and variables that Techstorm will call (besides some initialization). Do not directly call window code in here or your project will crash!

Same thing as a gate, but for game objects.

Represents an object that can be rendered in the screen. This is an abstract class that contains all the common properties of a game object.

You may implement this yourself to create your own game objects and drawing code, or you can inherit from Techstorm::GameObject to use the default implementation. However, if you inherit from this class, be sure you have a basic understanding of Raylib's and OpenGL's rendering system. If not, see https://learnopengl.com/ and https://www.raylib.com/ for a good place to start. In addition, it is not advised for you to inherit this yourself because it will require you to implement rendering code. However, it is recommended if your game requires a rendering system that is not provided by default.

The IGameObject class is abstract and cannot be instantiated directly. Instead, you should inherit from this class and implement the pure virtual functions. As of version 0.0.3, the pure virtual functions are: render(), texture(), update(), prePhysicsUpdate(), postUpdate(), destroy(), and cleanup().

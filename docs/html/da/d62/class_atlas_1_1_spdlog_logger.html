Detailed Description

A generic logger that uses the Spdlog library. According to the spdlog documentation, you may call the spdlog::log() function to log messages after the logger has been initialized because of how spdlog works. For more information, on how to use spdlog, please refer to their documentation. This class was designed to work in the same way as the multi-sink logger example provided in the spdlog documentation. Another important thing to note about this class is that, assuming you are not implementing a custom PFN_vkDebugUtilsMessengerCallbackEXT callback for Vulkan, this class should be initialized prior to the initialization and usage of Atlas' Vulkan API. Otherwise, you will get errors the second Vulkan starts sending messages to the callback.

To initialize this class in a case where you immediately have the initialization variables, consider the following code:

SpdlogLogger logger = SpdlogLogger("Atlas", "[Some Logger] [%^%l%$] %v", "log.txt");
logger.init();

If you do not have the initialization variables but need to set them later, you may make logger a pointer to get around the lack of a default constructor. However, keep in mind that a default constructor may be implemented later on.

Note
While most of the variables that are required for initialization within this class, there are a couple that must absolutely be set because there is no default value for them. More specifically, the values of the following variables that do not have a default value must be set PRIOR to calling init():
  • mMessageFormatPattern
  • mLogFilePath Please refer to the documentation of those variables for more information about them.

◆ initConsoleSink()

void Atlas::SpdlogLogger::initConsoleSink ( )
private

Initializes the console sink in the LoggerSinks struct that will be used by the spdlog logger. This function will set the console sink's log level to mConsoleLogLevel and will set the sink's pattern to mMessageFormatPattern. This is called by the init() function.

Note
Just because this function has been called, does not mean that the console sink is ready to be used because spdlog requires the console sink to be initialized before the spdlog::logger instance can be initialized. In the case of this class, the spdlog::logger instance is stored as mInternalSpdlogLoggerPtr. In other words, the variable mInternalSpdlogLoggerPtr must be initialized before any logging can be done.
Since
v0.0.1

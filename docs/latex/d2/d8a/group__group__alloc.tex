Maps memory represented by given allocation and returns pointer to it. 

Maps memory represented by given allocation to make it accessible to CPU code. When succeeded, {\ttfamily \texorpdfstring{$\ast$}{*}pp\+Data} contains pointer to first byte of this memory.

\begin{DoxyWarning}{Warning}
If the allocation is part of a bigger {\ttfamily Vk\+Device\+Memory} block, returned pointer is correctly offsetted to the beginning of region assigned to this particular allocation. Unlike the result of {\ttfamily vk\+Map\+Memory}, it points to the allocation, not to the beginning of the whole block. You should not add \doxylink{struct_vma_allocation_info_a4a3c732388dbdc7a23f9365b00825268}{Vma\+Allocation\+Info\+::offset} to it!
\end{DoxyWarning}
Mapping is internally reference-\/counted and synchronized, so despite raw Vulkan function {\ttfamily \doxylink{vulkan_8h_a5efbb24c826e19639a9bb7630ee8ed2b}{vk\+Map\+Memory()}} cannot be used to map same block of {\ttfamily Vk\+Device\+Memory} multiple times simultaneously, it is safe to call this function on allocations assigned to the same memory block. Actual Vulkan memory will be mapped on first mapping and unmapped on last unmapping.

If the function succeeded, you must call \doxylink{group__group__alloc_gaec4645c1787976e2b50d2dfd31279a2b}{vma\+Unmap\+Memory()} to unmap the allocation when mapping is no longer needed or before freeing the allocation, at the latest.

It also safe to call this function multiple times on the same allocation. You must call \doxylink{group__group__alloc_gaec4645c1787976e2b50d2dfd31279a2b}{vma\+Unmap\+Memory()} same number of times as you called \doxylink{group__group__alloc_ga13b93ec0ed38ad7232300cb94931c8d2}{vma\+Map\+Memory()}.

It is also safe to call this function on allocation created with \doxylink{group__group__alloc_ggad9889c10c798b040d59c92f257cae597a11da372cc3a82931c5e5d6146cd9dd1f}{VMA\+\_\+\+ALLOCATION\+\_\+\+CREATE\+\_\+\+MAPPED\+\_\+\+BIT} flag. Its memory stays mapped all the time. You must still call \doxylink{group__group__alloc_gaec4645c1787976e2b50d2dfd31279a2b}{vma\+Unmap\+Memory()} same number of times as you called \doxylink{group__group__alloc_ga13b93ec0ed38ad7232300cb94931c8d2}{vma\+Map\+Memory()}. You must not call \doxylink{group__group__alloc_gaec4645c1787976e2b50d2dfd31279a2b}{vma\+Unmap\+Memory()} additional time to free the "{}0-\/th"{} mapping made automatically due to \doxylink{group__group__alloc_ggad9889c10c798b040d59c92f257cae597a11da372cc3a82931c5e5d6146cd9dd1f}{VMA\+\_\+\+ALLOCATION\+\_\+\+CREATE\+\_\+\+MAPPED\+\_\+\+BIT} flag.

This function fails when used on allocation made in memory type that is not {\ttfamily HOST\+\_\+\+VISIBLE}.

This function doesn\textquotesingle{}t automatically flush or invalidate caches. If the allocation is made from a memory types that is not {\ttfamily HOST\+\_\+\+COHERENT}, you also need to use \doxylink{group__group__alloc_ga1059764930e85ac377a92b5237b5b919}{vma\+Invalidate\+Allocation()} / \doxylink{group__group__alloc_gab8093faf3afb114596d964d4bd24cb4a}{vma\+Flush\+Allocation()}, as required by Vulkan specification. \Hypertarget{group__group__alloc_ga29b210b1f39e49de844fc7b6944736d7}\label{group__group__alloc_ga29b210b1f39e49de844fc7b6944736d7}

\doxysubsection{Detailed Description}
Vulkan render interface for Rml\+Ui

My aim is to create compact, but easy to use class I understand that it isn\textquotesingle{}t good architectural choice to keep all things in one class But I follow to RMLUI design and for implementing one GAPI backend it just needs one class For user looks cool, but for programmer...

It\textquotesingle{}s better to try operate with very clean \textquotesingle{}one-\/class\textquotesingle{} architecture rather than create own library for Vulkan With many different classes, with not trivial signatures and etc And as a result we should document that library so it\textquotesingle{}s just a headache for all of us

Reminder to users\+: If you want to implement your Vulkan renderer check previous commits of this work, because current system works only with new and delete operations every frame (CPU side), on GPU we implemented the pre-\/allocated buffer with virtual allocs (Vma) so there\textquotesingle{}s no problems and all fine. I wrote all ideas and implementation for that.

\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename K, typename D, typename T$>$\newline
class Techstorm\+::abstract$<$ K, D, T $>$}
This is an {\bfseries{abstract interface singleton}} that all projects must implement and set the {\itshape \doxylink{class_techstorm_1_1abstract_1_1_project_reference}{Project\+Reference}} singleton to their project. If this boilerplate code is not done, \doxylink{namespace_techstorm}{Techstorm} will not know what to run. In terms of functionality, this interface just holds the functions and variables that \doxylink{namespace_techstorm}{Techstorm} will call (besides some initialization). {\bfseries{Do not directly call window code in here or your project will crash!}} 

Same thing as a gate, but for game objects.

Represents an object that can be rendered in the screen. This is an abstract class that contains all the common properties of a game object.

You may implement this yourself to create your own game objects and drawing code, or you can inherit from \doxylink{class_techstorm_1_1_game_object}{Techstorm\+::\+Game\+Object} to use the default implementation. However, if you inherit from this class, be sure you have a basic understanding of Raylib\textquotesingle{}s and Open\+GL\textquotesingle{}s rendering system. If not, see https\+://learnopengl.\+com/ and https\+://www.\+raylib.\+com/ for a good place to start. In addition, it is not advised for you to inherit this yourself because it will require you to implement rendering code. However, it is recommended if your game requires a rendering system that is not provided by default.

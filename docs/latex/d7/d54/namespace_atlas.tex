{\footnotesize\ttfamily Vk\+Bool32 Atlas\+::\+Default\+Vulkan\+Debug\+Callback (\begin{DoxyParamCaption}\item[{Vk\+Debug\+Utils\+Message\+Severity\+Flag\+Bits\+EXT}]{message\+Severity,  }\item[{Vk\+Debug\+Utils\+Message\+Type\+Flags\+EXT}]{message\+Type,  }\item[{const Vk\+Debug\+Utils\+Messenger\+Callback\+Data\+EXT \texorpdfstring{$\ast$}{*}}]{p\+Callback\+Data,  }\item[{\mbox{\hyperlink{imgui__impl__opengl3__loader_8h_ac668e7cffd9e2e9cfee428b9b2f34fa7}{void}} \texorpdfstring{$\ast$}{*}}]{p\+User\+Data }\end{DoxyParamCaption})}



The default Vulkan debug callback that \doxylink{namespace_atlas}{Atlas} uses if no custom debug callback is provided by the user in \doxylink{class_atlas_1_1_vulkan_instance_wrapper}{Vulkan\+Instance\+Wrapper} before it is initialized. According to the Khronos specification on the {\itshape PFN\+\_\+vk\+Debug\+Utils\+Messenger\+Callback\+EXT} object, this function should always return a value of {\itshape VK\+\_\+\+FALSE}, which is what this function does. See the Khronos specification for more information (this is in the see also section of the documentation). The message is logged via the usage of {\itshape spdlog\+::log()}. However, this means that some logger (more specifically a {\itshape Spdlog} logger) must have been initialized before this function is called or it will crash. 

\begin{DoxyRemark}{Remarks}
While most of these parameters are not used, they are included for consistency with the specifications that are listed in the Khronos specifications at the link in the {\bfseries{see also}} section.
\end{DoxyRemark}
\begin{DoxyPrecond}{Precondition}
A Spdlog logger must have been initialized before this function is called. This should be handled by the \doxylink{class_atlas_1_1_spdlog_logger}{Spdlog\+Logger} class.
\end{DoxyPrecond}

\hline

\hline
{\em p\+User\+Data} & A pointer to user data that was provided when the vulkan debug callback was set.\\

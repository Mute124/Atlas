\doxysection{Atlas\+::Spdlog\+Logger Class Reference}
\hypertarget{class_atlas_1_1_spdlog_logger}{}\label{class_atlas_1_1_spdlog_logger}\index{Atlas::SpdlogLogger@{Atlas::SpdlogLogger}}


A generic logger that uses the {\bfseries{Spdlog}} library. According to the spdlog documentation, you may call the {\itshape spdlog\+::log()} function to log messages {\bfseries{after}} the logger has been initialized because of how spdlog works. For more information, on how to use spdlog, please refer to their documentation. This class was designed to work in the same way as the multi-\/sink logger example provided in the spdlog documentation. Another important thing to note about this class is that, assuming you are not implementing a custom {\ttfamily PFN\+\_\+vk\+Debug\+Utils\+Messenger\+Callback\+EXT} callback for Vulkan, this class should be initialized prior to the initialization and usage of \doxylink{namespace_atlas}{Atlas}\textquotesingle{} Vulkan API. Otherwise, you will get errors the second Vulkan starts sending messages to the callback.

\doxysubsection{Detailed Description}
A generic logger that uses the {\bfseries{Spdlog}} library. According to the spdlog documentation, you may call the {\itshape spdlog\+::log()} function to log messages {\bfseries{after}} the logger has been initialized because of how spdlog works. For more information, on how to use spdlog, please refer to their documentation. This class was designed to work in the same way as the multi-\/sink logger example provided in the spdlog documentation. Another important thing to note about this class is that, assuming you are not implementing a custom {\ttfamily PFN\+\_\+vk\+Debug\+Utils\+Messenger\+Callback\+EXT} callback for Vulkan, this class should be initialized prior to the initialization and usage of \doxylink{namespace_atlas}{Atlas}\textquotesingle{} Vulkan API. Otherwise, you will get errors the second Vulkan starts sending messages to the callback. 

To initialize this class in a case where you immediately have the initialization variables, consider the following code\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\mbox{\hyperlink{class_atlas_1_1_spdlog_logger_a0d9050484259217f3c32081a0a4e3314}{SpdlogLogger}}\ logger\ =\ \mbox{\hyperlink{class_atlas_1_1_spdlog_logger_a0d9050484259217f3c32081a0a4e3314}{SpdlogLogger}}(\textcolor{stringliteral}{"{}Atlas"{}},\ \textcolor{stringliteral}{"{}[Some\ Logger]\ [\%\string^\%l\%\$]\ \%v"{}},\ \textcolor{stringliteral}{"{}log.txt"{}});}
\DoxyCodeLine{logger.init();}

\end{DoxyCode}


If you do not have the initialization variables but need to set them later, you may make {\ttfamily logger} a pointer to get around the lack of a default constructor. However, keep in mind that a default constructor may be implemented later on.

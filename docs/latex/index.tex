\doxylink{namespace_techstorm}{Techstorm} tries to be as easy to read as possible by making the code read similarly to English. In addition, \doxylink{namespace_techstorm}{Techstorm} tries to be intuitive and understandable at a quick glance, hence it uses Pascal\+Case and Camel\+Case. As such, the code conventions are simple, as outlined below.

Functions are simple, any that are not bound to a specific object will have the first letter lowercase. If it is a static, namespace, or global function, then it will have the first letter capitalized instead. When it comes to Preprocessor functions, it will follow the same rules as Constant Global variables, as seen below\+:

\begin{quote}
 \texorpdfstring{$\vert$}{|} Constant Global \texorpdfstring{$\vert$}{|} Cap \texorpdfstring{$\vert$}{|} Cap \texorpdfstring{$\vert$}{|} For this case, it will follow snake case {\bfseries{in all caps}} \texorpdfstring{$\vert$}{|} \end{quote}

{\footnotesize\ttfamily \mbox{\hyperlink{vulkan_8h_a459be4b1b89c972b0d6018bc483be9cc}{Vk\+Device\+Size}} Vma\+Allocation\+Info\+::offset}



Offset in {\ttfamily Vk\+Device\+Memory} object to the beginning of this allocation, in bytes. {\ttfamily (device\+Memory, offset)} pair is unique to this allocation. 

You usually don\textquotesingle{}t need to use this offset. If you create a buffer or an image together with the allocation using e.\+g. function \doxylink{group__group__alloc_gae9773fe0ef8582e23136d2d83c336ab9}{vma\+Create\+Buffer()}, \doxylink{group__group__alloc_gaa76142ae43535c862ad9e194a6e38aaa}{vma\+Create\+Image()}, functions that operate on these resources refer to the beginning of the buffer or image, not entire device memory block. Functions like \doxylink{group__group__alloc_ga13b93ec0ed38ad7232300cb94931c8d2}{vma\+Map\+Memory()}, \doxylink{group__group__alloc_ga7748e60ccc54e240777a848cb260bb19}{vma\+Bind\+Buffer\+Memory()} also refer to the beginning of the allocation and apply this offset automatically.

It can change after the allocation is moved during \doxylink{defragmentation}{Defragmentation}.

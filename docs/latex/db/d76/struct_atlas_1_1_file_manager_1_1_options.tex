How often should the {\ttfamily \doxylink{class_atlas_1_1_file_manager_1_1_file_janitor}{File\+Janitor}} should check for inactive files in {\bfseries{seconds}}. This is set to {\ttfamily ATLAS\+\_\+\+DEFAULT\+\_\+\+JANITOR\+\_\+\+CHECK\+\_\+\+INTERVAL} by default. Please refer to the note below for an important piece of advice regarding what to set this value to. 

\begin{DoxyNote}{Note}
As a piece of advice, this should be a multiple of {\ttfamily file\+TTL} to ensure that the file is not staying loaded for longer than {\ttfamily file\+TTL}. For example, if {\ttfamily file\+TTL} is 10 seconds, then {\ttfamily eviction\+Check\+Interval} should be set to 1, 2, 5 or 10 seconds. Once the file is inactive for 10 seconds, it will be unloaded right after the file\textquotesingle{}s lifespan is over, because {\ttfamily eviction\+Check\+Interval} is 10 seconds.
\end{DoxyNote}

{\footnotesize\ttfamily size\+\_\+t Atlas\+::\+File\+Manager\+::\+Options\+::max\+File\+Count \{ (size\+\_\+t)(-\/1)\}}



The maximum number of files that can be loaded at once. This is set to {\ttfamily -\/1} by default, which indicates there is no limit. If you want to set a limit, set this value to a positive number. 

\begin{DoxyNote}{Note}
If this limit is reached, the janitor will unload the least recently used file. This should be set to -\/1 for most cases, unless there is a specific reason to limit the number of files. This number should be set to a positive number if you want to limit the number of files.
\end{DoxyNote}

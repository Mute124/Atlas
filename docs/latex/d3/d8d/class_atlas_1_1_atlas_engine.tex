\doxysection{Atlas\+::Atlas\+Engine Class Reference}
\hypertarget{class_atlas_1_1_atlas_engine}{}\label{class_atlas_1_1_atlas_engine}\index{Atlas::AtlasEngine@{Atlas::AtlasEngine}}


This is the core and heart of \doxylink{namespace_atlas}{Atlas}; it is responsible for just about everything that is required to run the engine. It contains multiple different modules that can be used to extend the engine. If you want to extend the engine, you can simply inherit the respective module and utilize the concepts of polymorphism to your advantage. Keep in mind that the modules are stored as a \doxylink{class_atlas_1_1_thread_safe_variable}{Thread\+Safe\+Variable} and can be used concurrently. That means the user can multi-\/thread these modules if they so choose. This is just a default implementation of the \doxylink{class_atlas_1_1_i_atlas_engine}{IAtlas\+Engine} and you do not have to use this class. However, if you would like to use a different implementation, you can simply inherit from \doxylink{class_atlas_1_1_i_atlas_engine}{IAtlas\+Engine} and override the virtual functions. But one thing must remain\+: the class must inherit from \doxylink{class_atlas_1_1_i_atlas_engine}{IAtlas\+Engine} or be a child of \doxylink{class_atlas_1_1_i_atlas_engine}{IAtlas\+Engine}.
